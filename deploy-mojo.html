wildfly-extension:deploy

Full name:

org.jboss.plugins:wildfly-extension-maven-plugin:0.2-SNAPSHOT:deploy

Description:

Goal which deploys JBoss module to JBoss AS7/WildFly server

Attributes:

  • Requires a Maven project to be executed.
  • Binds by default to the lifecycle phase: install.

Required Parameters

Name Type Since Description
jbossHome File - Location of AS7/WildFly server to deploy to
Default value is: ${jboss.home}.
moduleZip File - Location of JBoss input module
Default value is: ${project.build.directory}/${project.artifactId}-module.zip.
serverConfig String - Location of server configuration file standalone.xml or domain.xml to write to (can be either relative to jbossHome or absolute)
Default value is: standalone/configuration/standalone.xml.
subsystem File - Location of subsystem content to be inserted into standalone.xml
Default value is: ${basedir}/src/main/resources/subsystem.xml.

Optional Parameters

Name Type Since Description
profiles String[] - List of server profiles to set subsystem on (only applies when serverConfig is domain.xml)
skipDeploy boolean - Whether to skip the execution of this mojo.
Default value is: false.

Parameter Details

jbossHome:

Location of AS7/WildFly server to deploy to
  • Type: java.io.File
  • Required: Yes
  • Default: ${jboss.home}

moduleZip:

Location of JBoss input module
  • Type: java.io.File
  • Required: Yes
  • Default: ${project.build.directory}/${project.artifactId}-module.zip

profiles:

List of server profiles to set subsystem on (only applies when serverConfig is domain.xml)
  • Type: java.lang.String[]
  • Required: No

serverConfig:

Location of server configuration file standalone.xml or domain.xml to write to (can be either relative to jbossHome or absolute)
  • Type: java.lang.String
  • Required: Yes
  • Default: standalone/configuration/standalone.xml

skipDeploy:

Whether to skip the execution of this mojo.
  • Type: boolean
  • Required: No
  • Default: false

subsystem:

Location of subsystem content to be inserted into standalone.xml
  • Type: java.io.File
  • Required: Yes
  • Default: ${basedir}/src/main/resources/subsystem.xml
